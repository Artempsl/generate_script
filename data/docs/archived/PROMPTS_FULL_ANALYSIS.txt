===============================================
AGENT ARCHITECTURE: STRICT SEQUENCE vs ReAct
===============================================

CURRENT IMPLEMENTATION:
-----------------------
This is a SIMPLIFIED ReAct - a strict linear pipeline with ONE conditional loop:

Flow:
  retrieve → web_search → synthesize → generate_outline → generate_script
    ↓
  validate → (if invalid) regenerate → validate (loop max 3x)
    ↓
  END

Characteristics:
✓ Fixed sequence of steps (no dynamic routing)
✓ ONE decision point: "should_regenerate" function
✓ No observation → reasoning → action loop
✓ No tool selection based on intermediate results
✓ No dynamic planning

FULL ReAct WOULD INCLUDE:
-------------------------
✗ Dynamic tool selection at each step
✗ Observation-based decision making
✗ Reasoning traces for "why choose this tool"
✗ Ability to skip/repeat arbitrary steps
✗ Backtracking to previous states
✗ Multiple conditional branches

WHY THIS DESIGN:
----------------
✓ Predictable execution for production use
✓ Easier to debug and monitor
✓ Guaranteed completion (no infinite loops)
✓ Clear token budget management
✓ Deterministic cost estimation
✓ Suitable for n8n integration (fixed workflow)

The current architecture is better described as:
"Linear Pipeline with Validation Loop" rather than "Full ReAct Agent"
